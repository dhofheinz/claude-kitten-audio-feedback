Installation (The Real Instructions)

Stop reading Medium articles about "10x engineers" and actually ship something. Here's how you get this working in 2 minutes:

1. Prerequisites (You Probably Have These)

Python 3.8+     # If you don't have this, what are you even doing?
Claude Code CLI # The actual Claude, not some knockoff
PulseAudio      # For sound on Linux/WSL2
~500MB disk     # For the model that actually works

2. Clone and Setup (Actually Works)

git clone 
cd 
./setup.sh      # Installs everything: venv, KittenTTS, MCP, the works
./test_audio.sh # Make sure you can hear the grizzled engineer

3. Enable Claude Integration

# For MCP (Claude speaks directly to you)
./setup_mcp.sh  # Shows you the exact command to run

# For hooks (audio on every file save)
cp .claude/settings.json.example .claude/settings.local.json
claude config set postToolUse /path/to/your/repo/.claude/review_code.py

4. Customize (Optional)

# Edit .env if you want different voices or toggles
cp .env.example .env  # (setup.sh does this for you)
vim .env  # Yes, vim. Deal with it.

# Key settings:
ENABLE_AUDIO_FEEDBACK=true  # Turn off when presenting
ENABLE_TEXT_FEEDBACK=true   # Turn off for silent mode
TTS_VOICE=expr-voice-2-m    # The grizzled engineer

Why This Doesn't Suck

The Grizzled Engineer Personality

Default voice is a 10x engineer who's been writing code since punch cards. Not condescending, just... experienced. Will tell you about that SQL injection vulnerability before it ruins your weekend. 

# What you'll hear:
"Kid, that variable name will haunt you at 3am"
"Seen this pattern before. Doesn't end well."
"That'll work... until production load hits"

Smart Text Splitting

KittenTTS has a ~400 char limit? No problem. We split at natural boundaries. Commas, conjunctions, sentences. Not mid-word like some amateur hour solution.

Rotating Logs (Because Disk Space Matters)

Logs rotate at 5MB. Keeps last 3. Configurable because one size doesn't fit all. Your 2TB NVMe doesn't care, but that EC2 instance might.

Kill Switches That Work

# Turn off audio when presenting
ENABLE_AUDIO_FEEDBACK=false

# Turn off text feedback when you just want silence
ENABLE_TEXT_FEEDBACK=false

# Because sometimes you need peace

Configuration (For Power Users)

Everything's in .env. No XML. No YAML. No TOML. Just key=value like nature intended.

The Important Bits

# Choose your personality
TTS_VOICE=expr-voice-2-m  # The grizzled one
TTS_VOICE=expr-voice-3-f  # Cheerful (why?)
TTS_VOICE=expr-voice-4-m  # Serious

# Batching (milliseconds)
BATCH_WAIT_TIME=3  # Wait 3s for more changes

# Model selection
TTS_MODEL=KittenML/kitten-tts-nano-0.1  # Fast and good

# Audio player
AUDIO_PLAYER=paplay  # PulseAudio (Linux/WSL2)
AUDIO_PLAYER=afplay  # macOS
AUDIO_PLAYER=ffplay  # When nothing else works

The Technical Stuff (If You Care)

Architecture That Makes Sense

PostToolUse hooks ‚Üí Claude analysis ‚Üí Queue tips ‚Üí Background processor ‚Üí Parallel audio generation ‚Üí Your ears. No microservices. No message buses. Just Unix processes doing one thing well.

Security (Because 2024)

JSON via stdin, not command args. File locks prevent races. Subprocess isolation. No eval(). No string interpolation in commands. The basics that everyone should do but doesn't.

Performance

15M parameters = ~60MB RAM. Generation in ~200ms. Adaptive polling (100ms ‚Üí 500ms ‚Üí 1s) to save CPU. ThreadPoolExecutor for parallel chunks. It's not rocket science, just competent engineering.
